📊 Key Relationships Summary

Relationship Type Description
Company → Subscription 1:1 Each company has one subscription
Subscription → SubscriptionPlan N:1 Multiple subscriptions can use same plan
User → Role N:1 Multiple users can have same role
WorkOrder → Tasks/Labors/Costs 1:N One work order has many tasks, labors, costs
PreventiveMaintenance → WorkOrder 1:N PM schedule generates multiple work orders
Asset → Location N:1 Multiple assets can be at same location
Team ↔ Users M:M Teams contain multiple users, users can be in multiple teams
Part ↔ Assets M:M Parts can be used in multiple assets
Meter → Reading 1:N One meter has many readings over time
TaskBase → TaskOption 1:N Task template has multiple options
CustomField → CustomFieldValue 1:N One field definition has many values across entities
