1. Project Overview

What This Project Does

This is a CMMS (Computerized Maintenance Management System) - a digital tool that helps companies manage all their maintenance work.

Real-world example: Imagine a factory with 100 machines. When a machine breaks down, someone needs to:
  • Record the problem
  • Assign a technician to fix it
  • Track what parts were used
  • Record how long it took
  • Calculate the cost
  • Keep history for future reference
This application does all of that digitally through a web interface.

4.5 Exception Handling

// Custom exception with HTTP status
public class CustomException extends RuntimeException {
    private final HttpStatus httpStatus;

    public CustomException(String message, HttpStatus status) {
        super(message);
        this.httpStatus = status;
    }
}

// Global handler catches all exceptions
@RestControllerAdvice
public class GlobalExceptionHandlerController {

    @ExceptionHandler(CustomException.class)
    public ResponseEntity<SuccessResponse> handleCustomException(...) {
        return new ResponseEntity<>(
            new SuccessResponse(false, ex.getMessage()),
            ex.getHttpStatus()
        );
    }
}

4.6 Security (JWT)

// 1. User logs in
@PostMapping("/signin")
public ResponseEntity<AuthResponse> login(@RequestBody UserLoginRequest req) {
    String token = userService.signin(req.getEmail(), req.getPassword(), req.getType());
    return ResponseEntity.ok(new AuthResponse(token));
}

// 2. JWT Token creation
public String createToken(String username, List<RoleType> roles) {
    Claims claims = Jwts.claims().setSubject(username);
    claims.put("auth", roles.stream()...);  // Add roles to token
    return Jwts.builder().setClaims(claims)...compact();
}

// 3. Token validated on each request
public class JwtTokenFilter extends OncePerRequestFilter {
    protected void doFilterInternal(HttpServletRequest req, ...) {
        String token = jwtTokenProvider.resolveToken(req);
        if (token != null && jwtTokenProvider.validateToken(token)) {
            Authentication auth = jwtTokenProvider.getAuthentication(token);
            SecurityContextHolder.getContext().setAuthentication(auth);
        }
    }
}

9. Beginner Mistakes to Avoid

Based on this project's patterns, here are common mistakes to avoid:

1. Not Using Transactions Properly

// ❌ WRONG - Multiple DB operations without transaction
public void updateWorkOrder(Long id, WorkOrderDTO dto) {
    WorkOrder wo = repo.findById(id).get();
    wo.setTitle(dto.getTitle());
    repo.save(wo);  // What if next line fails?
    notificationService.notify(dto.getAssignee());  // Might fail!
}

// ✅ CORRECT
@Transactional
public void updateWorkOrder(Long id, WorkOrderDTO dto) {
    // All or nothing
}

2. Returning Entities from Controllers

// ❌ WRONG
@GetMapping("/{id}")
public WorkOrder getById(@PathVariable Long id) {
    return workOrderService.findById(id).get();  // Exposes DB structure!
}

// ✅ CORRECT
@GetMapping("/{id}")
public WorkOrderShowDTO getById(@PathVariable Long id) {
    WorkOrder wo = workOrderService.findById(id).get();
    return workOrderMapper.toShowDto(wo);  // Return DTO
}

3. N+1 Query Problem

// ❌ WRONG - Causes N+1 queries
List<WorkOrder> orders = repo.findAll();
for (WorkOrder order : orders) {
    order.getAssignedTo().size();  // Each triggers a DB query!
}

// ✅ CORRECT - Use JOIN FETCH
@Query("SELECT w FROM WorkOrder w LEFT JOIN FETCH w.assignedTo")
List<WorkOrder> findAllWithAssigned();

4. Hardcoding Values

// ❌ WRONG
if (status.equals("COMPLETE")) { ... }

// ✅ CORRECT
if (status.equals(Status.COMPLETE)) { ... }
// Or better - use enums properly
if (status == Status.COMPLETE) { ... }

5. Not Checking Permissions

// ❌ WRONG
@DeleteMapping("/{id}")
public void delete(@PathVariable Long id) {
    workOrderService.delete(id);  // Anyone can delete!
}

// ✅ CORRECT
@DeleteMapping("/{id}")
@PreAuthorize("hasRole('ROLE_ADMIN')")
public void delete(@PathVariable Long id, HttpServletRequest req) {
    User user = userService.whoami(req);
    if (!user.canDelete(workOrder)) {
        throw new CustomException("Forbidden", HttpStatus.FORBIDDEN);
    }
}

6. Ignoring LazyInitializationException

// ❌ WRONG - Accessing lazy collection outside transaction
WorkOrder order = service.findById(id);  // Transaction ends
order.getFiles().size();  // BOOM! LazyInitializationException

// ✅ CORRECT
@Transactional
public List<File> getFiles(Long id) {
    WorkOrder order = repo.findById(id).get();
    return order.getFiles();  // Still in transaction
}

7. Not Using Optional Properly

// ❌ WRONG
User user = repo.findById(id).get();  // Throws if not found!

// ✅ CORRECT
User user = repo.findById(id)
    .orElseThrow(() -> new CustomException("Not found", HttpStatus.NOT_FOUND));

Summary & Next Steps

Key Takeaways:
  • Multi-tenant SaaS architecture (companies separated)
